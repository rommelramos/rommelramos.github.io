3.1.1 LINUX (PPGBM)

Link para a sala do classroom para os estudantes do PPGBM: Classe Google ClassRoom

3.1.1.1 Orientações

  • Terminal - Aplicativo, geralmente relacionado a ausência de interface gráfica, de onde se pode executar comandos;
  • Diretório corrente - Diretório onde se está acessando via terminal no momento;
  • Diretório raiz - Diretório / que é o diretório de onde são originados os demais
  • Home- Faz referência ao diretório do usuário, caso o usuário seja "fulano", o home é /home/fulano

  • * é um símbolo coringa, por exemplo: *.txt, significa que se busca algo que tenha qualquer prefixo mas que o sufixo seja .txt

  • Sintaxe básica da execução de comandos no terminal

  • NomeDoComando [opções] [parâmetros] - Sintaxe de comandos

3.1.1.2 Estrutura de Diretórios

    Nível 0 - /
    Nível 1 - /home
    Nível 2 - /home/user
    Nível 3 - /home/user/arquivo.txt
    Nível 3 - /home/user/diretorio1
    Nível 4 - /home/user/diretorio1/arquivo2.txt

3.1.1.3 Indentificando o diretório onde se está

pwd- Verificar o caminho onde se está no terminal

    mkdocs.yml    # The configuration file.
    docs/
        index.md  # The documentation homepage.
        ...       # Other markdown pages, images and other files.

3.1.1.4 Acessando diretórios

cd permite navegar até determinada pasta.

Use cd .. (com dois pontos seguidos) para retornar ao diretório imediatamente anterior

Use cd ~ para acessar o diretório inicial de outro usuário.

3.1.1.5 Listando arquivos e diretórios

ls Lista todos os arquivos e diretórios dentro do diretório onde se está executando ou no diretório alvo.

ls Comando simples

ls /home/rommelramos Comando simples indicando o diretório alvo

ls -R Lista todos os arquivos nos subdiretórios.

ls -a Lista todos os arquivos ocultos.

ls -lha Lista todos com informações detalhadas, incluindo permissões, data de criação e tamanho de arquivos.

3.1.1.6 Criando Arquivos

touch arquivo.txt Criando um arquivo vazio

echo "conteudo de teste" > arquivo.txt Direcionando a saíde do comando echo para um arquivo através do símbolo >

3.1.1.7 Criando diretórios

mkdir dir1 Criando diretório dentro do diretório corrente.

mkdir /home/rommel/dir1 Criando diretório dentro do diretório home do usuário Rommel.

3.1.1.8 Visualizando conteúdo de arquivos

less arquivo.txt Visualizando no terminal o conteúdo do arquivo.txt. Pressionando o espaço, enter ou direcional para baixo e para cima consegue nevagar pelo arquivo.

more arquivo.txt Visualizando no terminal o conteúdo do arquivo.txt. Pressionando o espaço caso o texto tenha muitas linhas ele será percorrido.

cat arquivo.txt Visualizando no terminal o conteúdo do arquivo.txt.

3.1.1.9 Copiando arquivos e Diretórios

cp [arquivo1] [arquivoN] [destino]

cp arquivo.txt /home/rommel/Documents Copiando o arquivo.txt para o diretório /home/rommel/Documents

cp arquivo1.txt arquivo2.txt arquivo3.txt /home/username/Documents- Copiando o arquivo1.txt, arquivo2.txt e arquivo3.txt para o diretório /home/rommel/Documents

cp -r [diretório] [destino]

cp dir1/ /home/rommel/Documents Copiando o diretório dir1/ para dentro do diretório /home/rommel/Documents

3.1.1.10 Removendo Arquivos e Diretórios

rm [arquivo1]

rm arquivo.txt Removendo arquivo

rm -rf dir1/ Removendo diretório

3.1.1.11 Movendo Arquivos e Diretórios

mv [arquivo1] dirnovo/[arquivot_novo.txt]

mv arquivo1.txt /home/rommelramos/Documents/- Movendo arquivo1.txt para diretório /home/rommelramos/Documents/

mv arquivo1.txt arquivo_novo.txt- Renomeando o arquivo1.txt para arquivo_novo.txt

3.1.1.12 Tornando-se super usuário

sudo su - Caso seu usuário esteja no arquivo /etc/sudoers

        root ALL = (ALL) ALL

Adicionando o usuário rommel ao sudoers

        root ALL = (ALL) ALL
        rommel ALL = (ALL) ALL

find dir1/ -name *.txt- Procura no diretório dir1/ qualquer arquivo com sufixo *.txt

locate index.html - Buscando o arquivo index.html no sistema em todos os diretórios a partir do banco de dados.
