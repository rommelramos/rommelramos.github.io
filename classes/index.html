3.1.1.1 Orientações

1 Terminal - Aplicativo, geralmente relacionado a ausência de interface gráfica, de onde se pode executar comandos;
2 Diretório corrente - Diretório onde se está acessando via terminal no momento;
3 Diretório raiz - Diretório / que é o diretório de onde são originados os demais;
4 Home- Faz referência ao diretório do usuário, caso o usuário seja "fulano", o home é /home/fulano ;
5 * é um símbolo coringa, por exemplo: *.txt, significa que se busca algo que tenha qualquer prefixo mas que o sufixo seja .txt*

  • Sintaxe básica da execução de comandos no terminal

  • NomeDoComando [parâmetros] [argumentos] - Sintaxe de comandos

Parâmetro = Variável que vai receber o valor

Argumento = Valor que será atribuído

3.1.1.2 Como saber a localização do arquivo

Alguns programas podem ser executados a partir de qualquer diretório que você esteja, pois eles estão no $PATH, uma variável de ambiente que aponta para diretóris que contém arquivos e programas.

which nomeprograma Comando recebe como argumento apenas o nome do programa

3.1.1.3 Estrutura de Diretórios

    Nível 0 - /
    Nível 1 - /home
    Nível 2 - /home/user
    Nível 3 - /home/user/arquivo.txt
    Nível 3 - /home/user/diretorio1
    Nível 4 - /home/user/diretorio1/arquivo2.txt

O diretório /home/user , onde user é o seu usuário pode ser chamando de HOME

3.1.1.4 Indentificando o diretório onde se está

pwd- Verificar o caminho onde se está no terminal

3.1.1.5 Acessando diretórios

cd permite navegar até determinada pasta.

Use cd .. (com dois pontos seguidos) para retornar ao diretório imediatamente anterior

Use cd ~ para acessar o diretório inicial de outro usuário.

3.1.1.6 Listando arquivos e diretórios

ls Lista todos os arquivos e diretórios dentro do diretório onde se está executando ou no diretório alvo.

ls Comando simples

ls /home/rommelramos Comando simples indicando o diretório alvo

ls -R Lista todos os arquivos nos subdiretórios.

ls -a Lista todos os arquivos ocultos.

ls -lha Lista todos com informações detalhadas, incluindo permissões, data de criação e tamanho de arquivos.

3.1.1.7 Criando Arquivos

touch arquivo.txt Criando um arquivo vazio

echo "conteudo de teste" > arquivo.txt Direcionando a saíde do comando echo para um arquivo através do símbolo >

printf "Rommel\nRamos\n" > arquivo.txt Permite o uso de caracteres especiais

Você também pode redirecionar a saída de comandos:

ls > "resultado_ls.txt" #Teste e observe o resultado visulizando o resultado

3.1.1.8 Visualizando conteúdo de arquivos

less arquivo.txt Visualizando no terminal o conteúdo do arquivo.txt. Pressionando o espaço, enter ou direcional para baixo e para cima consegue nevagar pelo arquivo.

more arquivo.txt Visualizando no terminal o conteúdo do arquivo.txt. Pressionando o espaço caso o texto tenha muitas linhas ele será percorrido.

cat arquivo.txt Visualizando no terminal o conteúdo do arquivo.txt.

3.1.1.9 Criando diretórios

mkdir dir1 Criando diretório dentro do diretório corrente.

mkdir /home/rommel/dir1 Criando diretório dentro do diretório home do usuário Rommel.

3.1.1.10 Copiando arquivos e Diretórios

cp [arquivo1] [arquivoN] [destino]

cp arquivo.txt /home/rommel/Documents Copiando o arquivo.txt para o diretório /home/rommel/Documents

cp arquivo1.txt arquivo2.txt arquivo3.txt /home/username/Documents- Copiando o arquivo1.txt, arquivo2.txt e arquivo3.txt para o diretório /home/rommel/Documents

cp -r [diretório] [destino]

cp -r dir1/ /home/rommel/Documents Copiando o diretório dir1/ para dentro do diretório /home/rommel/Documents

3.1.1.11 Filtrando arquivos com pistas

grep '[pista]' [arquivo alvo]

grep 'teste' arquivo.txt Este comando retornará apenas as listas que apresentam a palavra teste

grep "^u" /etc/passwd Recuperando do arquivo passwd todas as linhas que INICIAM pela letra u

grep "s$" /etc/passwd Recuperando do arquivo passwd todas as linhas que TERMINAM pela letra u

3.1.1.12 Criando Sessoões Screem

screen -S [nome da sessão] #Cria a sessão com o nome definido
screen -ls #Lista todas as sessões screen do usuário
screen -r [id ou nome da sessão] # Carregar o screen disponível
screen -D [id ou nome da sessão] # Desconectar o screen acessado

#dentro do screen
CTRL+a+c #Criar uma sub-sessão no screen
CTRL+a+p #Retorna nas sessões do screen
CTRL+a+n #Avança nas sessões do screen

3.1.1.13 Manipulando arquivos

wc -l [arquivo] #conta a quantidade de linhas do arquivo
head -n [número de linhas] [arquivo] # Extrai o [número de linhas] do arquivo de cima para baixo
tail -n [número de linhas] [arquivo] # Extrai o [número de linhas] do arquivo de baixo para cima

Editando o um arquivo

vim arquivo.txt

comandos vim:

Tecla i - Colocar em modo de insernção SHIFT+ : Entrar em modo de prompt

Salvar arquivo e sair: em modo de prompt, pressionar ESC então wq! Sair arquivo sem salvar: em modo de prompt q!

sudo apt update #Atualiza a lista de pacotes e programas que podem ser instalados na máquina.

sudo apt upgrade #Atualiza o sistema, baixa e instala as atualizações dos pacotes

Exemplo

       deb http://security.ubuntu.com/ubuntu focal-security main restricted
       deb http://us.archive.ubuntu.com/ubuntu focal-updates main restricted

3.1.1.21 Buscando/Instalando/removendo Pacotes do Sistema operacional

apt-cache search pista #Localiza pacotes disponíveis para o sistema no repositório
apt-get install pacote10.2 #Com o nome do pacote identificado, você pode usar o NOME EXATO DO PACOTE para instalar
apt-get remove pacote10.2 #Com o nome do pacote identificado, você pode usar o NOME EXATO DO PACOTE para instalar

3.1.1.22 Levantando um Serviço SSH

apt install openssh-server

vim /etc/ssh/ssh_config #Apenas com permissões você pode alterar o arquivo

# Deve procurar a indicação da porta de acesso e alterar a porta padrão 22 para outra disponível
AddressFamily any
ConnectTimeout 0
StrictHostKeyChecking ask
IdentityFile ~/.ssh/id_rsa
IdentityFile ~/.ssh/id_dsa
IdentityFile ~/.ssh/id_ecdsa
IdentityFile ~/.ssh/id_ed25519
Port 22 #Lista correta, trocar o número 22 por outro.

3.1.1.23 Acesso remoto via Secure Socket Shell (SSH)

ssh [usuário]@[IP] -p [Porta de Acesso]

Servidor: 200.239.101.158 Porta: 22

ssh rommel@200.239.101.158 -p 22

Copiando arquivo remoto

scp [IP origem]:[caminho do arquivo ou diretório] [destino local]

Copiando arquivo Remoto para minha máquina. 

scp biotec@200.239.92.159:/home/biotec/arquivo.txt . # O ponto **.** significa o diretório corrente. Copiando o arquivo remoto para o diretório local


scp arquivo.txt biotec@200.239.92.159:/home/biotec #Copiando o arquivo do diretório local, e corrente, para o servidor remoto.

Para copiar diretórios, deve-se utilizar o parâmetro -r 

 scp -r dir1/ biotec@200.239.92.159:/home/biotec

md5sum arquivo.txt O argumento é apenas o arquivo para verificar a integridade de arquivos após transmissão

Resultado da execução:

1ca308df6cdb0a8bf40d59be2a17eac1  arquivo.txt
